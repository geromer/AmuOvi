Community Sports

Run your sporting event from start to finish.
Read time ~15 minutes · For admins and public users

Community Sports is a web platform built to manage community sporting events — from athlete registration and team formation through to live results, leaderboards, and printed reports.

It has two types of users: the public (athletes and spectators who view events and scores) and admins (organizers who manage everything).

At a glance

🗓️

Event management

Create Track, Field, and Team Game events with age-group schedules and real-time status tracking.

🏃

Athlete profiles

Register athletes with auto-assigned bib numbers, age group classification, and team assignment.

🏆

Live leaderboard

Points calculated automatically after every result entry. Live scores pushed via WebSocket.

📄

Reports

Marshalling sheets, heat sheets, team game entry reports, standings, roster, and more — all printable.

⚙️

Sport config

Configure points per position for each sport type, age calculation date, and registration settings.

✅

Data validation

Audit report detects overlapping age groups, unassigned athletes, missing results, and leaderboard mismatches.

01Accessing the app

The app is a web application — open it in any modern browser.

Navigate to the app URL provided by your event organizer. No installation is required. The app works on desktop and mobile browsers.

User typeWhat they can do
Public View events, schedules, results, and the leaderboard — no login required.
Admin Manage all data: athletes, teams, events, results, reports, and configuration. Requires login.

02Public pages

No login needed. Everything a spectator or athlete needs to follow the action.

🗓️

Events

Browse all events — Track, Field, and Team Game. Each event card shows the sport type, category, and current status (Upcoming Ongoing Completed). Tap an event to see its schedule and results.

⏱️

Event schedules

Each Track or Field event has age-group schedules showing start time, end time, and location. Team Game events have a single schedule. Status updates automatically as results are entered.

📊

Results

View recorded performances per event — athlete name, bib number, measurement, position, and points awarded. Team game results show team name, squad label, position, and points.

🏆

Leaderboard

Two views: per-event (which team earned what points in a specific event) and overall (cumulative points across all events, ranking teams from first to last).

03Logging in as admin

Admins log in from the Login page to access the management interface.

  1. Go to the Login pageClick the Login link from any page, or navigate to /login.
  2. Enter your email and passwordThese are set by the system administrator. Contact your organizer if you don't have credentials.
  3. Click Sign InYou are redirected to the admin dashboard with access to all management pages.
Public registration If registration is enabled (controlled by an admin in Sport Config), a "Create account" link appears on the login page. Athletes can self-register with basic details. Admins can disable this at any time.
Session length Admin sessions expire after 24 hours. You will be asked to log in again after that.

04Age groups

Age groups define the competition categories athletes are placed into.

Go to Age Groups in the admin sidebar. Each age group has:

FieldDescription
LabelDisplay name — e.g. U14 Boys, Open Women
Min age / Max ageDecimal ages (e.g. 12.0–14.0). Range is exclusive lower bound, inclusive upper bound: an athlete aged exactly 12.0 does NOT qualify; 14.0 does.
GenderMALE, FEMALE, or MIXED
Bib range start / endOptional. If set, athlete bib numbers are assigned sequentially within this range (e.g. 001–025). Once exhausted, overflow uses suffix A (026A). If not set, the fallback format ATH-XXXX is used.

Creating an age group

  1. Click New Age GroupThe create form opens in the panel on the right.
  2. Fill in the label, min age, max age, and genderOptionally set a bib range if you want sequential bib numbers for this group.
  3. Click SaveThe age group appears in the list immediately.
Overlapping age ranges Make sure age group ranges do not overlap — the Data Validation report will flag overlaps. Overlaps cause an athlete to match multiple groups, which is not allowed.
Deleting age groups An age group cannot be deleted if any athlete is assigned to it, or if any event schedule references it. Reassign or remove those first.

05Teams

Teams group athletes together and earn collective leaderboard points.

Go to Teams in the admin sidebar. Each team has a name, an optional abbreviation, and a payment status (Pending / Paid / Waived).

Creating a team

  1. Click New TeamA small create form opens at the top of the list.
  2. Enter the team name and abbreviationAbbreviation is used in compact table views.
  3. Click CreateThe team is saved and appears in the list.
Athletes are assigned to teams separately After creating a team, go to the Athletes page to assign individual athletes to it. A team can have any number of athletes; athletes can only belong to one team at a time.
Deleting teams A team cannot be deleted if any athlete is a member of it, or if any team entry (squad) exists for it in an event. Remove those associations first.

06Athletes

Athletes are the individual competitors. Each athlete is linked to a user account and an age group.

Go to Athletes in the admin sidebar. The list shows every athlete with their bib number, age group, team, and payment status.

Creating an athlete

  1. Click New AthleteThe create form expands at the top of the list.
  2. Enter personal detailsFull name, email, date of birth, and phone number. An athlete account is created automatically.
  3. Select age group and teamAge group determines which events the athlete is eligible for. A bib number is assigned automatically based on the age group's bib range.
  4. Set payment statusPending, Paid, or Waived.
  5. Click SaveThe athlete appears in the list with their assigned bib number.
Bib numbers are permanent A bib number is assigned once at creation and never changes — even if the athlete's age group is later updated. This is intentional so printed materials stay accurate.
Age calculation Age is calculated as a decimal (e.g. 14.9) using the Age Calculation Date set in Sport Config — not today's date. The formula is days elapsed ÷ 365.25, floored to 1 decimal place. This date is set once before the event and used consistently for all athletes.

Editing an athlete

Click the pencil icon on any athlete row. The edit form opens inline. You can update any field except the bib number. Click Save to confirm.

Deleting athletes An athlete cannot be deleted if they have any recorded result with a position or points. Remove their results first. Deleting an athlete also deletes their linked login account.

07Events

Events are the competitions. Each event has a sport type that determines how it is scheduled and how results are recorded.

Go to Events in the admin sidebar.

Sport typeDescriptionResults
TRACK Running events — 100m, 200m, 400m, etc. Individual athlete performance per age group
FIELD Field events — long jump, shot put, discus, etc. Individual athlete performance per age group
TEAM GAME Team-vs-team events — relay, tug of war, etc. Team squad results (no age group on entry)

Creating an event

  1. Click New EventThe create form opens in the panel.
  2. Enter a titleE.g. "100m Sprint", "Long Jump", "Relay Race".
  3. Select the sport typeTRACK, FIELD, or TEAM_GAME. This cannot be changed after creation.
  4. Enter the categoryA free-text label for grouping — e.g. "Sprint", "Throws", "Finals".
  5. Click SaveThe event is created. Next, set up its schedule.
Deleting events An event cannot be deleted if it has any age-group schedules assigned, or any team entries (squads). Remove those first.

08Schedules — Track & Field

For TRACK and FIELD events, you assign one or more age groups to the event, each with its own timing and location.

Click on a TRACK or FIELD event in the Events list to open its detail page. The Schedules panel shows the age groups assigned so far.

Assigning an age group

  1. Click Add ScheduleThe schedule form opens.
  2. Select an age groupEach age group can only be assigned once per event.
  3. Set start time, end time, and locationThese appear on the public event page and on printed call sheets.
  4. Click SaveThe schedule is added. Status starts as UPCOMING and updates automatically when results are recorded.
Status is automatic for Track & Field When the first result is recorded for an event + age group, the schedule status becomes ONGOING. When results are cleared (reset), it reverts to UPCOMING. Status is set to COMPLETED when a full result set is recorded. You can also manually override status from the schedule row.
Deleting a schedule A schedule cannot be deleted if any athlete has a recorded result for that event + age group combination. Clear results first via the Results page.

09Team Game events

Team Game events work differently — teams enter as squads, and there are no individual age-group schedules.

Click on a TEAM_GAME event in the Events list. You will see two panels:

📅

Event Schedule panel

A single schedule for the whole event — start time, end time, and location. Create this first by clicking Add Schedule. There is only one schedule per Team Game event.

👥

Team Entries panel

Each team competing in the event needs a team entry (squad). After adding a team entry, you assign individual athletes to that squad. This is where you build the roster for each team game event.

Setting up a Team Game event — step by step

  1. Create the event scheduleIn the Event Schedule panel, click Add Schedule and set the timing and location.
  2. Add team entriesIn the Team Entries panel, click Add Team Entry. Select a team and enter a squad label (e.g. "Squad A"). Repeat for each competing team.
  3. Assign athletes to each squadClick the squad card to expand it, then click Add Athlete. A search modal lets you find athletes by name. Add as many as needed for that squad.
  4. Record results when the event runsSee section 10 — Results.
Age groups and Team Game Team Game events do not have per-age-group schedules. An athlete's age group comes from their athlete profile, but it does not affect which squad they can join. Squad assignment is managed entirely from the Event page.
Deleting a team entry A team entry (squad) cannot be deleted after a result (position) has been recorded for it. The delete button is also disabled in the UI once a position exists.

10Recording results

Results are entered from the Results page in the admin sidebar.

Track & Field results

Results are recorded per athlete, per event, per age group.

  1. Go to Results → Athlete ResultsSelect the event and age group from the dropdowns.
  2. Enter performance for each athleteRecord the measurement (e.g. 12.34 seconds, 6.80 metres), unit, and position.
  3. Click SavePoints are calculated automatically from the position using the Sport Config points table for that sport type. The leaderboard is rebuilt immediately.
Tied positions If two athletes share a position (e.g. both 2nd), the app averages the points for positions 2 and 3, then applies the floor — both athletes receive the same rounded-down average. Position 3 is skipped.
Clearing a result To remove an athlete's result, open their entry in the Results page and click Clear. If no athlete in that event + age group has any remaining result, the schedule status reverts to UPCOMING.

Team Game results

Results are recorded per team entry (squad), not per individual athlete.

  1. Go to Results → Team ResultsSelect the Team Game event from the dropdown. All team entries for that event appear.
  2. Assign a position to each team entryEnter the finishing position for each squad.
  3. Click SavePoints are calculated from the position using the Team Game points table in Sport Config. The leaderboard is rebuilt.
Schedule status for Team Games Status updates automatically: if some squads have positions recorded and others don't, the schedule becomes ONGOING. Once all squads have a position, it becomes COMPLETED. If all positions are cleared, it reverts to UPCOMING.

11Leaderboard

Points accumulate in real time as results are recorded.

The leaderboard is visible to everyone — no login required. There are two views:

📊

Per-event leaderboard

Shows each team's points earned in a single event — sum of all athlete performances (Track/Field) or team entry results (Team Game) for that event. Accessible from the event detail page.

🏆

Overall leaderboard

Ranks all teams by total points across every event. This is the main competition standings. Updates live after every result save.

Live updates The leaderboard is pushed to all connected browsers in real time via WebSocket when results are recorded. Spectators watching on their phones see the update without refreshing.

12Reports

All reports are in the Reports page — admin only. Each report opens as a printable modal.

ReportWhat it shows
Athlete Numbers List Full list of athletes with their bib numbers, age group, and team. Useful for producing numbered bibs.
Club Athlete Roster Athletes filtered by team. Shows name, bib, age group, and payment status. Filter by teamId.
Top Performers Athletes ranked by total points within an age group. Filter by ageGroupId.
Age Group Athlete Summary Per-age-group breakdown of athlete counts per event. Includes a filter to hide events below a threshold count.
Team Overall Standings Full standings table — each team's total points across all events, ranked.
Athlete Marshalling & Call Sheet Groups athletes by sport type, gender, and age group for marshalling. Filter by sport type, gender, or specific age groups.
Heat Sheet / Entry Report Randomised heat lane assignments. TRACK: Position × Heat grid. FIELD: Round 1–6 table per athlete. Each age group prints on its own page. A "Re-randomize" button reshuffles without re-fetching.
Team Game Entry Report One page per Team Game event — squad list, score column, position column, a Finals placement grid (1st–4th), and a signatures row. No bib numbers.
Data Validation Report Six-section audit: age group overlaps, athletes without age groups, events without schedules, empty schedules, leaderboard mismatches, and orphaned entries. See section 14.

Printing a report

  1. Open the Reports pageClick Reports in the admin sidebar.
  2. Click the report buttonA modal opens with filters and a preview.
  3. Apply any filtersMost reports have dropdowns or toggles to narrow the data.
  4. Click PrintYour browser's print dialog opens. Choose "Save as PDF" to keep a file, or send to a printer.

13Sport config

Central settings for how the competition is scored and managed.

Go to Sport Config in the admin sidebar (or click the gear icon). Settings are grouped by category.

General settings

SettingWhat it does
Age Calculation Date The reference date used to calculate every athlete's decimal age. Set this once before the competition starts and do not change it mid-event — all age group placements depend on it.
Public Registration Toggle whether the "Create account" link appears on the login page. When disabled, only admins can create new user accounts.

Points tables

There is a separate points table for each sport type: Track, Field, and Team Game. Each table maps a finishing position (1st, 2nd, 3rd…) to a number of points.

Example

Track points

Position 1 → 10 pts, Position 2 → 8 pts, Position 3 → 6 pts, Position 4 → 5 pts, and so on.

Note

Changes apply to new results only.

Updating a points table does not retroactively change already-recorded results. If you need to recalculate, use the Data Validation → Rebuild Leaderboard tool.

Certificate coordinates You can configure the x/y positions (in mm) for where rank labels are placed on printed certificates — one set of coordinates per rank (1st, 2nd, 3rd). This is used by the certificate print feature if enabled.

14Data validation

The Validation report is a full audit of your event data — run it before and during the competition to catch problems early.

Go to Reports → Data Validation Report. The report runs six checks:

CheckWhat it flags
Age group overlaps Any two age groups whose decimal age ranges overlap. Uses exclusive lower bound logic (minAge, maxAge]. Overlapping groups mean an athlete could fall into two categories.
Athletes without age groups Athletes whose assigned age group has been deleted or is missing. These athletes cannot be placed in any event.
Events without schedules Events that have no age-group schedules (Track/Field) or no team event schedule (Team Game).
Schedules with no athletes Age-group schedules where no athlete in that age group exists.
Leaderboard mismatches Cases where a team's leaderboard total doesn't match the sum of their recorded performance or team entry points (tolerance: 0.01). Indicates a calculation or data bug.
Orphaned entries Results or team entries that reference deleted events, age groups, or athletes.

Rebuild Leaderboard

If the validation report shows leaderboard mismatches, click Rebuild Leaderboard. This re-aggregates all points from the source tables (athlete_performance and team_entry) into the leaderboard. It fixes aggregation bugs but does not recalculate individual point values.

Manual trigger only The rebuild does not run automatically on page load. An admin must click the button deliberately to apply it.

15User management

Admins can view and manage all user accounts from the Users page.

Go to Users in the admin sidebar. The full user list loads with client-side filtering:

Filter

Search by name or email

Free-text search across full name and email address. Results update as you type. Searching resets the letter filter.

Filter

Role toggles

Pill buttons for each role — ADMIN, ATHLETE, SPECTATOR. Toggle any on or off. Default view excludes ATHLETEs to reduce noise.

Filter

A–Z letter pagination

Filter by the first letter of the user's full name. Only letters with matching users are enabled.

Note

Athlete accounts

When an athlete is created on the Athletes page, a linked user account is created automatically. Deleting the athlete also deletes the account.

Roles

RoleAccess
ADMIN Full read and write access to all data, reports, and configuration.
ATHLETE Can log in but has limited access — primarily used to associate a person with an athlete profile.
SPECTATOR Authenticated but no special access beyond the public pages. Useful for giving someone a login without admin rights.
Only ADMINs can write data All create, edit, and delete operations on any entity require the ADMIN role. Athletes and Spectators can only view the same public pages available to unauthenticated visitors.
